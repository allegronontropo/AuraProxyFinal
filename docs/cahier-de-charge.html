2. Contexte et problematique

2.1 Contexte metier

L'essor des modeles de langage (LLM) tels que OpenAI GPT, Anthropic Claude et Mistral a transforme le developpement d'applications intelligentes. Cependant, les entreprises font face a trois defis majeurs :

DefiDescriptionImpact
Couts imprevisiblesFacturation au token sans plafondFactures surprises de milliers d'euros
Vendor lock-inChaque fournisseur impose son format d'APICout de migration eleve
Absence de visibilitePas de vue unifiee sur consommation et coutsDecisions sans donnees

2.2 Problematique

Comment offrir aux entreprises une couche intermediaire entre leurs applications et les fournisseurs d'IA, garantissant le controle des couts, l'interchangeabilite des fournisseurs et la visibilite en temps reel ?

2.3 Solution proposee

Aura Proxy est un middleware intelligent situe entre les applications clientes et les fournisseurs d'IA :

10. Design Patterns

PatternConcept NestJSApplicationBenefice
StrategyProviders @Injectable() + DIChaque fournisseur LLM implemente LLMProviderAjouter un fournisseur = 1 classe
Decorator@UseInterceptors()Logging, suivi des couts, retryPas de modification du code metier
Singleton@Injectable() + @Global()PrismaService et RedisServicePas de fuite de connexions
Observer@nestjs/event-emitter + @OnEvent()Alertes, metriques, notificationsDecouplage emetteur / recepteur

Strategy Pattern — Providers LLM

// Interface commune — tous les providers l'implementent
interface LLMProvider {
  readonly name: ProviderName;
  chat(request: ChatRequest): Promise<ChatResponse>;
  stream(request: ChatRequest): AsyncIterable<StreamChunk>;
  estimateCost(usage: TokenUsage): number;
}

// Enregistres comme services injectables NestJS
@Injectable()
export class OpenAIProvider implements LLMProvider { ... }

@Injectable()
export class AnthropicProvider implements LLMProvider { ... }

Decorator Pattern — Interceptors

@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler) {
    const start = Date.now();
    return next.handle().pipe(
      tap(() => this.logger.log(`Traitee en ${Date.now() - start}ms`)),
    );
  }
}

// Application declarative sur le controller
@UseInterceptors(LoggingInterceptor, CostTrackerInterceptor, RetryInterceptor)
@Controller('v1')
export class ChatController { ... }

Observer Pattern — Evenements

// Emission (dans BudgetGuard)
this.eventEmitter.emit('budget.exceeded', { projectId, used, limit });

// Ecoute (dans AlertListener)
@OnEvent('budget.exceeded')
async handleBudgetExceeded(payload: BudgetExceededEvent) {
  await this.discord.send(`Budget depasse: ${payload.projectId}`);
}

14. Flux applicatifs

15.1 Requete proxy complete

sequenceDiagram
    actor Client
    participant AG as AuthGuard
    participant BG as BudgetGuard
    participant RL as RateLimiter
    participant Cache as CacheService
    participant Chat as ChatService
    participant Prov as LLM Provider
    participant Events as EventEmitter

    Client->>AG: POST /v1/chat/completions
    AG->>AG: SHA-256 lookup DB/Redis
    alt Cle invalide
        AG-->>Client: 401 Unauthorized
    end
    AG->>BG: ApiKeyPayload
    BG->>BG: Redis GET budget
    alt Budget depasse
        BG->>Events: emit budget.exceeded
        BG-->>Client: 429 BUDGET_EXCEEDED
    end
    BG->>RL: Budget OK
    RL->>RL: Redis ZADD + ZCARD
    alt Rate limit depasse
        RL-->>Client: 429 + Retry-After
    end
    RL->>Cache: Rate OK
    Cache->>Cache: embedding + pgvector search
    alt Cache HIT
        Cache-->>Client: 200 + x-cache HIT
    end
    Cache->>Chat: Cache MISS
    Chat->>Prov: SDK call
    Prov-->>Chat: ChatResponse
    Chat->>Cache: store response
    Chat->>Events: emit request.completed
    Chat-->>Client: 200 + x-cache MISS

15.2 Flux streaming SSE

sequenceDiagram
    actor Client
    participant Proxy as NestJS Proxy
    participant Prov as LLM Provider
    participant TC as TokenCounter
    participant BS as BudgetService

    Client->>Proxy: POST stream=true
    Proxy->>Proxy: Set headers SSE
    Proxy->>Prov: provider.stream(request)
    loop Chaque chunk
        Prov-->>Proxy: StreamChunk
        Proxy->>TC: countTokens
        TC-->>Proxy: runningTotal
        alt Cout > budget restant
            Proxy-->>Client: error budget_mid_stream
        else OK
            Proxy-->>Client: data delta content
        end
    end
    Prov-->>Proxy: Stream termine
    Proxy-->>Client: data [DONE]
    Proxy->>BS: recordSpend
